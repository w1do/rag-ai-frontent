# SUMMARY

## Current Status
- Разработка интерактивных компонентов для ИИ-ассистента и RAG-системы.
- Интегрирована базовая логика диалога с API `http://localhost/share-chat/2/message`.

## Recent Changes
- Создан компонент `ChatHero.astro` для быстрой интеграции чата в Hero-секцию.
- Создан компонент `AIChat.astro` — полнофункциональный классический чат-интерфейс для ИИ-ассистента (клиент задаёт вопрос — бот отвечает).
- Стилизация `AIChat.astro` выполнена в фирменной палитре сайта через CSS-переменные темы (`--primary-color` #DFFF00, `--background-one`, тёмный фон) со scoped-стилями (Tailwind в проекте не подключён).
- Реализован полный UX чата: аватары собеседников, анимированный индикатор печати, плавное появление сообщений, авто-скролл, обработка ошибок сети.
- Реализована поддержка авто-изменения размера текстового поля и отправки по Enter (Shift+Enter — новая строка).
- Добавлена страница `src/pages/chat.astro` (маршрут `/chat`) с секцией-заголовком и встроенным `AIChat`, JSON-LD `WebApplication`.
- Запрос к ассистенту: `POST {apiBase}/share-chat/{chatId}/message` с телом `{ question }`, парсинг поля `answer`; эндпоинт, `chatId` и тексты вынесены в props.
- Интегрированы SEO-ключи (RAG, ИИ-ассистент, LLM) в структуру компонентов.
- Исправлена ошибка «Не удалось связаться с ассистентом»: нормализован `apiBase` (убран двойной слэш в URL `share-chat/{chatId}/message`), добавлены таймаут запроса (30 с) и логирование реальной причины ошибки в консоль.
- Иконки человечка/робота в сообщениях заменены на текстовые подписи: «Я» (сообщения пользователя) и «Ассистент» (ответы бота); добавлена структура `.ai-chat__content`/`.ai-chat__name`.
- Индикатор ожидания ответа теперь подписан «Ассистент печатает» с пульсирующей анимацией (`ai-chat-pulse`) поверх анимированных точек.
- Подписи отправителей оформлены бейджиками: «Я» — сероватый (полупрозрачный белый), «Ассистент» — жёлтый (`--primary-color`); индикатор «Ассистент печатает» теперь пульсирует текстом в рамке с расходящимся свечением (`box-shadow`).
- Исправлено отсутствие бейджиков: стили `AIChat` переведены в `<style is:global>`, т.к. scoped-стили Astro не применяются к создаваемым в JS элементам сообщений.
- Ответы ассистента теперь рендерятся из Markdown в HTML (заголовки, списки, **жирный**/*курсив*, `код`, блоки ```code```, ссылки, цитаты) с экранированием HTML; сообщения пользователя и ошибки остаются простым текстом. Добавлены стили markdown внутри `.ai-chat__bubble`.
- Чат переписан на React: подключена интеграция `@astrojs/react`, добавлен компонент `src/components/Shared/react/AIChat/AIChat.tsx` с co-located стилями `AIChat.css` (FSD-структура); состояние, fetch с таймаутом 30 с, авто-resize textarea, отправка по Enter и индикатор «печатает» реализованы через хуки. Markdown рендерится библиотекой `marked`. Старый `AIChat.astro` удалён, компонент подключён в `Hero.astro` и `chat.astro` через `client:visible`.
- Исправлено «чат не отображается на index»: причина — устаревший кэш Vite (`node_modules/.vite`) в dev-сервере, запущенном до добавления интеграции React, из-за чего React-остров падал при гидратации с ошибкой `_jsxDEV is not a function` и блок чата оставался пустым. Кэш `node_modules/.vite` и `.astro` очищены; после перезапуска dev-сервера чат корректно рендерится в Hero на `/` (проверено сборкой и визуально через Playwright). Код компонента менять не потребовалось.
- Добавлена инициализация чата через `GET {apiBase}/share-chat/{chatId}/init`: при монтировании компонент получает `welcome_message` (заменяет стартовое приветствие), список `actions` и данные компании (`company_name`, `description`, `phone`). Шапка чата теперь показывает название компании, описание и кликабельный телефон (`tel:`). Под историей сообщений выводятся кнопки быстрых действий — клик отправляет соответствующий вопрос. `send` доработан для приёма текста напрямую (без устаревшего замыкания draft). Добавлены стили `.ai-chat__phone`, `.ai-chat__actions`, `.ai-chat__action`.
